BendinGrass ChangeLog

v2.2:
[Unreleased, pending review]

  • Conversion tool
    BendinGrass now include a TerrainData conversion tool! You can use it to convert terrains if you are using an Unity version that does not support shader overriding on grass (Unity 2019.2.x).
  • Universal Rendering Pipeline
    Included a shader for URP! Use the Terrain Conversion tool to set the correct shaders, if needed.
  • Grass inflate
    Added a checkbox to prevent grass from "inflating".
  • Flat color
    You can now chose a different color (and alpha) for bent grass!
  • Compatibility
    You can now disable BendinGrass on specific graphic APIs.

v2.1:
01/04/2019

  • Billboards
    Added support for billboard grass, as requested by users.
  • Editor
    Added a new tab in the volume inspector, in case you need to change how the volume is displayed inside the editor.
    You can change color or disable single features.
  • Trees
    Benders can now interact with trees!

    I decided to ship tree support with this update, even though I'm not really happy with the end result.
    It will probably be improved in future updates.
    You can tweak or disable tree interactions in the inspector under the Graphics Parameters section.

  • Minor changes
    Several very minor changes related to the inspector or code.
  • Bugfixes
    Multiple bug fixes on this update, with the most important ones being:
    Fixed a problem that could cause a volume map glitch on moving volumes under certain circumstances.
    Fixed a bug that would cause the volume map gizmos to be drawn incorrectly in the editor.
    Fixed a possible cause of shader compiling error on some configurations.
    Included a fix in case of incorrect depth rendering of the grass.

    Use it if you have problems with rendering (for example on camera effects). To use it, you have to go in your project settings, under the Graphics tab, then set BendinGrass/BendinGrass-DepthNormalsTexture as replacement for "Depth Normals".

v2.0:
30/06/2018

  • Space Partitioning
    You can now split your big volume in multiple smaller ones, for a massive performance boost
  • Dynamic Volume Simulation
    Only compute the relevant part of your map, saving much computation power!
    You can now move your simulation volume around and only simulate a small part of your level
  • Terrain mode
    If your level uses one terrain you can now use its heightmap for better space-efficiency
  • Benders inheritance
    It can be useful sometime to chain grass benders and inherit parameters. You can now do that!
  • More compatible
    Even tho BendinGrass is meant to be used on high end platforms, it now works on a wider range of devices. I got rid of all the DX11 level functions and also designed my own rendering system to replace the Unity's custom render textures. It works on Shader Level 3.0+ platforms:
    • DX9
    • DX10
    • OpenGLES 2
    • OpenGLES 3
    So you can now use it on WebGL or mobile.
  • More capable
    No more hard limit on the number of benders.
    The renderer can now split the bendmap rendering into multiple draw calls if necessary, to avoid some GPU memory limit.
